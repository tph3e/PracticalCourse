\subsection{Batch Allocation}\label{subsec:batch}
\textit{(Can Güleç, B)}\quad
In the batch allocation engine, a threshold-based batching strategy was implemented for resource allocation. We place the new created tasks in an internal buffer until a predetermined number of tasks (the batch size, in our implementation $k_{\text{limit}} = 5$) is reached, instead of allocating resources to tasks immediately upon creation. Once there are enough tasks in the buffer, the engine attempts to allocate resources to the entire batch of tasks.

During batch allocation, each task is allocated to the first available resource capable of performing the required activity. Once allocated, a given resource cannot be allocated again within the same batch. This ensures that simultaneous allocations to the same resource are avoided. If a task cannot be allocated (because there are no suitable resources available), it remains in the buffer and will be reconsidered during the next allocation cycle.

To ensure that no tasks remain unprocessed at the end of the simulation, there is a "flush" function included in the engine that forcefully allocates all remaining buffered tasks regardless of whether or not the batch size has been met.
