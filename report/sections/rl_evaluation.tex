\subsubsection{Reinforcement-Learning Policy}\label{ssub:rl-eval}
\textit{(Aldo Patrone)}\quad
The RL policy (Sec.~\ref{subsec:rl}) is evaluated against reference baselines on a
data-calibrated case-based environment (arrival rate and case length matched to
BPIC-17), scored on the disclosed equal-weight combined objective of cycle time and
fairness (Table~\ref{tab:alloc_cmp}, plotted in the Appendix
Fig.~\ref{fig:alloc_cmp_plot}). Both RL agents learn sensible trade-offs (REINFORCE
leans towards cycle time, PPO towards fairness) and clearly beat the random,
round-robin and experience-greedy baselines. However, \emph{shortest-queue remains
the best}: load-balancing serves both objectives at once (fairness directly and cycle
time by keeping resources available), so it is a remarkably strong baseline that
neither agent surpasses. The honest takeaway is that well-chosen heuristics are hard
to beat for this objective. An RL edge would require a richer state and process model,
or a harder objective, together with the extensive training the literature uses. This
is the standalone comparison against reference baselines. The cross-method evaluation
of all group methods on the integrated simulator is done in Sec.~\ref{ssub:cross-eval}.
